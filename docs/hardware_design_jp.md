# ATtiny85 コンバットハードウェア設計

これは、再構築されたATtiny85 Combatボードの詳細なハードウェア設計図です。

## 目標

- ATtiny85 RESET/PB5 をリセットとして保持する。
- 通常のGPIOピンを5つだけ使用してください。
- ボタンを5つサポートします。
- 5×4のLEDマトリックスに対応。
- 通常の7セグメントLED数字を2桁までサポートします。
- パッシブブザーを1つサポートします。
ディスプレイのスキャンピンをボタンやブザーと共有しないようにしてください。

## 主要アーキテクチャ

74HC595シフトレジスタを2個、デイジーチェーン接続して使用します。

- U2: マトリックス列および7セグメントセグメント用のLEDアノードバス。
- U3: マトリックス行と7セグメント数字用のローサイド選択制御。
- U3の出力はLEDのカソードを直接駆動するのではなく、NPNトランジスタのベースを駆動します。

ディスプレイは6つのフェーズに多重化されています。

1. 行列の0行目
2. 行列の1行目
3. 行列の2行目
4. 行列の3行目
5. 7セグメント数字0
6. 7セグメント数字1

一度に選択されるのは、1行または1桁のみです。

## MCUピン割り当て

| ATtiny85 ピン | 信号 | 宛先 |
|---|---|---|
| PB0 |シフト_SER | U2 DS/SER |
| PB1 | SHIFT_CLK | U2 SHCP/SRCLK および U3 SHCP/SRCLK |
| PB2 | SHIFT_LATCH | U2 STCP/RCLK および U3 STCP/RCLK |
| PB3 | BTN_ADC | 5ボタンADCラダー |
| PB4 | ブザー | パッシブブザードライバー |
| PB5 | リセット | リセット専用、GPIOとしては使用されません |

## 74HC595 ピン接続

SN74HC595、TC74HC595、またはその他の3V対応HCファミリー部品を使用してください。
3 V では HCT ファミリー部品の使用を避けてください。

## ローサイド選択ドライバー

マトリックスの各行および7セグメントの各桁ごとに、NPNトランジスタを1個使用します。

推奨部品：

- 2N3904
- 2SC1815
- BC547
- S8050

各選択トランジスタについて：

| ノード | 接続 |
|---|---|
| ベース | U3 出力は 4.7k まで |
| エミッタ | GND |
| コレクタ | 選択されたカソードバス |

任意だが推奨：

- リセット時にトランジスタをオフにするため、各トランジスタのベースからGNDまで100kΩの抵抗を挟む。

## LEDマトリックス配線

列に陽極、行に陰極を配置した5×4のマトリックスを使用してください。

| マトリックス信号 | 接続 |
|---|---|
| COL0 アノード | BUS0 |
| COL1 アノード | BUS1 |
| COL2 アノード | BUS2 |
| COL3 アノード | BUS3 |
| COL4 アノード | BUS4 |
| ROW0 カソード | Q_ROW0 コレクタ |
| ROW1 カソード | Q_ROW1 コレクタ |
| ROW2 カソード | Q_ROW2 コレクタ |
| ROW3 カソード | Q_ROW3 コレクタ |

電流制限抵抗は、U2出力とBUS0～BUS4の間にある330Ω抵抗アレイです。

## 2桁7セグメント配線

共通陰極型7セグメントディスプレイを使用してください。

| セグメント  | 接続 |
|----|--------|

| c  | BUS2   |
| d  | BUS3   |
| e  | BUS4   |

| g  | BUS6   |
| dp | BUS7、オプション |
| 数字 0 共通カソード | Q_DIG0 コレクタ |
| 数字 1 共通カソード | Q_DIG1 コレクタ |

電流制限抵抗は、U2出力とBUS0～BUS7の間にある330Ω抵抗アレイです。

## ボタンADCラダー

入力ボタンとしてPB3のみを使用してください。

推奨始動回路：

- PB3/ADCノードは10kΩを介して+3Vにプルアップされます。
各ボタンは、異なる抵抗器を介してPB3/ADCとGNDを接続します。

推奨抵抗値：

ボタン | GND への直列抵抗 |
|---|---|
| 右 | 0オーム |
| ダウン | 1.0k |
| 上昇 | 2.2k |
| 左 | 4.7k |
| ファイヤー | 10k |

10kΩのプルアップ抵抗を用いた場合のADC値の予測値はあくまで概算値であり、測定が必要です。
実際の盤上で。

読み取り値にノイズが多い場合は、PB3/ADCとGNDの間に小さなコンデンサを追加してください。
推奨される初期値：1 nF～10 nF。

## ブザードライバー

PB4を専用のブザー出力として使用します。

推奨回路：

| ノード | 接続 |
|---|---|
| PB4 | 1kΩ抵抗器からNPNベースへ |
| NPNエミッタ | GND |
| NPNコレクター | ブザーネガティブ |
| ブザープラス | +3V |

完成図面では、電磁式パッシブブザー9042_16R、ローサイドドライバー
2SC1815、フライバックダイオード1N4148を使用します。

| 部品端子 | 接続 |
|---|---|
| 1N4148カソード（帯のある側） | +3V |
| 1N4148アノード | NPNコレクタ／ブザー負極 |

1N4148はブザーと並列に接続し、直列には入れません。通常動作中は
逆バイアスとなり、トランジスタをOFFにしたときの逆起電力を抑えます。

## 電力とデカップリング

追加：

- ATtiny85のVCC/GND付近に0.1μFのセラミックコンデンサ。
- 各74HC595のVCC/GND付近に0.1μFのセラミックコンデンサを配置。
- U2/U3付近の+3VとGNDの間に10μFの電解バルクコンデンサを配置。
  プラス端子を+3V、マイナス端子をGNDへ接続します。

コイン型電池で駆動する場合、多重化のためディスプレイの明るさが低下する可能性があります。
LEDのピーク電流は電圧降下を引き起こします。単4電池または安定化電源を使用する方が安全です。

## ソフトウェアの極性

U2バスのビットはアクティブハイです。

- バスビット1：セグメント/カラムアノードが有効。
- バスビット0：セグメント/カラムオフ。

U3の選択ビットは、NPNドライバの手前でアクティブハイになります。

- U3選択ビット1：対応する行/桁が選択されます。
- U3選択ビット0：対応する行/桁がオフ。

更新ルーチンは次のことを行う必要があります。

1. U3の選択ビットをすべてオフにし、U2バスをすべてオフにします。
2. ラッチをかける。
3. 目的のU2バスパターンとU3の選択ビットを1つシフトします。
4. ラッチをかける。
5. 次の更新ティックで次のフェーズに進みます。

これにより、シフトレジスタの更新中に発生するゴースト現象を回避できます。

## 2桁表示の動作

U4は10の桁、U5は1の桁です。待機中は得点を表示し、通常プレイ中は
`00`から始まるゼロ埋めのアップカウントを表示します。敵を倒した時は
約500msだけ得点を表示し、次のステージ開始後はカウント表示へ戻ります。
ゲームオーバー時は最終スコアを保持します。
